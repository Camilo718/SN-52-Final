# Proyecto SN-52: Aplicación del Personal Software Process (PSP)

## Sobre este proyecto

Este repositorio documenta la aplicación de la metodología del Proceso de Software Personal (PSP) en el desarrollo del proyecto SN-52. El objetivo es mejorar el rendimiento y la calidad del software a través de un enfoque disciplinado y basado en datos.

## Objetivos del PSP en SN-52

La implementación del PSP en este proyecto busca:

-   Mejorar las habilidades de estimación y planificación de las tareas.
-   Establecer compromisos de entrega realistas y cumplirlos.
-   Gestionar de manera proactiva la calidad del proyecto.
-   Reducir el número de defectos en el producto final.

## Estructura del PSP Aplicada

El proyecto seguirá un enfoque evolutivo, adoptando los niveles del PSP de manera progresiva:

#### PSP0, PSP0.1 (Disciplina y Medición del Proceso)

-   **Fases:** Planificación, Desarrollo (diseño, codificación, compilación, pruebas) y Post-mortem.
-   **Medición Base:** Se registrará el tiempo invertido, los defectos inyectados/removidos y el tamaño del código para establecer una línea base del proceso actual.
-   **Mejora:** Se definirá un estándar de codificación y se creará un Plan de Mejora del Proceso (PIP) con ideas para optimizar el flujo de trabajo.

#### PSP1, PSP1.1 (Estimación y Planificación)

-   **Estimación:** Utilizando los datos de la fase anterior, se estimará el tamaño y el tiempo total para las nuevas funcionalidades del proyecto.
-   **Seguimiento:** Se registrará el tiempo real invertido en cada tarea para refinar futuras estimaciones y planificar el cronograma del proyecto.

#### PSP2, PSP2.1 (Gestión de Calidad y Diseño)

-   **Nuevas Fases:** Se incorporarán revisiones de diseño (Design Review) y revisiones de código (Code Review).
-   **Enfoque en Calidad:** El objetivo principal será la prevención y eliminación temprana de defectos, midiendo el tiempo y los defectos por fase.
-   **Herramientas:** Se utilizarán checklists personalizadas para las revisiones de diseño y código.

## Recolección de Datos y Métricas Clave

La recopilación de datos es fundamental. Se utilizarán las siguientes métricas para analizar y mejorar el proceso:

-   **Tamaño:** Medido en líneas de código (LOC).
-   **Esfuerzo:** Tiempo dedicado a cada tarea, registrado en minutos.
-   **Calidad:** Número de defectos encontrados en el producto.
-   **Cronograma:** Seguimiento del progreso comparando fechas planificadas con las reales.

Estos datos se registrarán de manera consistente para permitir un análisis estadístico que ayude a mejorar la planificación y la calidad.

## Gestión de la Calidad

El objetivo es entregar un software de alta calidad con un número mínimo de defectos. La estrategia se centra en:

-   **Detección Temprana:** Las fases del PSP están diseñadas para encontrar defectos lo antes posible, reduciendo el tiempo en fases posteriores como las pruebas.
-   **Revisiones Estructuradas:** Se realizarán revisiones personales de diseño y código utilizando checklists que se adaptarán para enfocarse en los errores más comunes identificados a lo largo del proyecto.

Este enfoque disciplinado asegura que los defectos se eliminen cerca de donde se introdujeron, mejorando la eficiencia y la calidad del producto final.
